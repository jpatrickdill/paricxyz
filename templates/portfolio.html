Roblox Requests

Docs GitHub

Roblox Requests is an HTTP library I created for Roblox. It's a wrapper to Roblox's HttpService.

I created Requests because Roblox's HttpService is difficult to use, not readable, and leaves a lot of work to be done for the developer.

Roblox Requests was my first time creating an API in Lua. I wanted to create a library that was simple, effective, and easy to understand, so I modeled it after the Python Requests library that I use often.

This library also gave me my first experience creating a documentation website from scratch, using the Jekyll static site generator. The website can be seen at requests.paric.xyz.

Features:

  • Sessions with Cookie Persistence & Default Headers
  • Automatic Query Strings
  • JSON Body Encoding, Response Decoding
  • HTML/XML Parsing with DOM, jQuery Selectors
  • Key/Value Cookies
  • Multipart File Encoding and Upload
  • Custom Per-Session Ratelimiting

Roads Plugin

This is a tool I created for modeling roads in Roblox for driving games.

Two common ways to model roads in Roblox are:

  1. Manually using basic geometries, either sticking to a grid or using an arc plugin to curve it, leaving overlapping textures
  2. Using a plugin to make a manually built road model follow a Bezier curve, giving smoother roads, but still leaving overlapping parts/textures

While both of these methods work, they certainly don't maximize efficiency. I created my road plugin as an alternative.

It features an editor that that uses "nodes" to layout a road path. Each lane of traffic is considered independent, allowing you to refactor roads and easily create forks, merges, slip lanes, etc. while only rendering a single surface where lanes are grouped.

Publishing this plugin was my first time marketing something I made. I used Adobe Photoshop to create thumbnails for the plugin and Adobe Premiere Pro to create a trailer video as well as tutorials.

The plugin has been installed 30,000 times since its initial release in May 2020.

RbxServerList

Many Roblox communities exist on Discord, whether it's a few people playing with their friends or entire game audiences. RbxServerList is a website and Discord bot I created that simplifies sharing private servers within these communities.

Instead of manually maintaining a list of private servers like many communities do, RbxServerList allows members of your Discord to "donate" their private server.

Every 30 seconds, RbxServerList checks every private server in its pool, getting player counts and ensuring that join links are still active. Users are also allowed to choose max player counts, which will hide their private server from the list when reached.

The two major challenges I faced creating RbxServerList were validating private servers and keeping updated player counts. Because Roblox didn't have public API endpoints for either of these, I had to figure it out myself using Chrome devtools to watch the network as I browsed server lists from the Roblox website.

RbxServerList was my first time using OAuth2 on a public project, as donating a serer online requires signing in via Discord. The website uses Redis to store live information for each private server.
